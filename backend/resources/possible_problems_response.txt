This modified BPMN model includes gateways to indicate relationships between tasks. The Exclusive Gateway is used to determine whether the buyer's credit is approved or rejected. If the credit is approved, the process moves to the Task to fulfill the order. If the credit is rejected, the process moves to the Intermediate Catch Event to cancel the order. The Parallel Gateway is used to indicate that the Tasks to send the invoice and fulfill the order can be done in parallel. Once both Tasks are completed, the process moves to the End Event. This modification ensures that the process is optimized for efficiency and can handle different scenarios.
